# Identity and least privilege

## Principles

- Least privilege always  
- Explicit identity for all actions  
- No shared high-privilege accounts  

## Requirements

- Use managed identity or delegated access  
- Align permissions to user context  
- Restrict access to required systems only  

## Risks

- Over-privileged tools  
- Identity confusion  
- Uncontrolled actions  

## Rule

Agents must never have more access than the user they represent.
