# Glossary

Use this page to understand key terms used in this guidance.

---

# Core concepts

## Agent

An AI system that can:

- Interpret inputs  
- Generate outputs  
- Take actions using tools  

---

## Agentic AI

AI systems that can:

- Plan tasks  
- Perform multi-step actions  
- Interact with tools and systems  

---

## Autonomy

The level of independence an agent has.

Types:

- Read-only  
- Drafting (requires approval)  
- Tool-using  
- Actioning  

---

## Human in the loop (HITL)

A human reviews or approves outputs before actions are taken.

---

## Human in command

Humans retain final decision authority at all times.

---

# Patterns

## RAG (Retrieval-Augmented Generation)

Retrieves relevant documents before generating responses.

---

## Tool-using agent

An agent that calls APIs or systems to perform tasks.

---

## Workflow agent

An agent embedded in a structured process.

---

## Planner → executor

An agent that breaks tasks into steps and executes them.

---

## Multi-agent system

Multiple agents working together with defined roles.

---

# Integration

## MCP (Model Context Protocol)

A standard for agents to:

- Call tools  
- Access APIs  
- Perform actions  

---

## Tool

An external system or API used by an agent.

---

## API

A system interface used to exchange data.

---

## AI Gateway

A control layer that manages:

- Authentication  
- Logging  
- Policy enforcement  

---

# Security and data

## Prompt injection

An attack that manipulates an agent’s behaviour.

---

## Data exfiltration

Unauthorised access or transfer of data.

---

## Least privilege

Access limited to what is strictly required.

---

## Data minimisation

Using only the data necessary for a task.

---

# Governance

## DPIA

Data Protection Impact Assessment.

Required when processing personal data.

---

## ATRS

Algorithmic Transparency Recording Standard.

Describes how AI systems are used.

---

## ALB

Arm’s Length Body (e.g. Environment Agency, Natural England).

---

# Delivery

## SaaS

Software provided as a managed service.

---

## Low-code

Build using configuration with minimal coding.

---

## Pro-code

Full custom development.

---

## Lifecycle

Stages of delivery:

- Discovery  
- Alpha / Beta  
- Live  
- Retirement  

---

# Monitoring and evaluation

## KPI

Key Performance Indicator.

Measures success.

---

## Red-teaming

Testing systems under adversarial conditions.

---

## Drift

Change in model performance over time.

---

# Summary

You must understand:

- What agents are  
- How they behave  
- How they are controlled  

> Clear definitions help prevent misuse and reduce risk.
