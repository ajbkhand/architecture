# Secure agent systems

Use this page to apply **security controls to AI agents**.

Agents introduce new risks beyond traditional systems.

---

# What you must do

You must:

- ✅ Apply security guardrails  
- ✅ Control agent behaviour  
- ✅ Protect data and systems  
- ✅ Monitor continuously  

---

# Key risks

Agent systems introduce:

- Prompt injection  
- Data leakage  
- Tool misuse  
- Uncontrolled actions  

---

# Control autonomy

You must:

- ✅ Define autonomy level  
- ✅ Require human approval for actions  
- ✅ Prevent unrestricted behaviour  

---

⚠️ Never allow unrestricted autonomy  

---

# Secure tool access

You must:

- ✅ Apply least privilege  
- ✅ Restrict tool access  
- ✅ Validate all inputs  
- ✅ Log all actions  

---

# Protect data

You must:

- ✅ Minimise data use  
- ✅ Protect sensitive data  
- ✅ Apply encryption  
- ✅ Control outputs  

---

# Monitor and log

You must:

- ✅ Log all interactions  
- ✅ Monitor behaviour  
- ✅ Detect misuse  
- ✅ Respond to incidents  

---

# Test before deployment

You must:

- ✅ Test for prompt injection  
- ✅ Test for misuse  
- ✅ Validate behaviour  
- ✅ Run red-teaming  

---

# Apply lifecycle controls

You must:

- Identify risks early  
- Design controls during build  
- Test before deployment  
- Monitor in production  

---

# Do

- ✅ Design for failure  
- ✅ Apply layered controls  
- ✅ Monitor continuously  

---

# Do not

- ❌ Assume agents are safe  
- ❌ Allow uncontrolled actions  
- ❌ Deploy without testing  

---

# Summary

You must:

1. Control behaviour  
2. Protect systems and data  
3. Test before deployment  
4. Monitor continuously  

> Agent security is about controlling what systems can do, not just what they say.
